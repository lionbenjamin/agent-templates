# Clarify Skill

Review specifications for clarity and completeness.

## When to Activate

This skill is relevant when:
- Reviewing product specifications
- Checking for ambiguity or gaps
- Validating feature scope and feasibility
- Ensuring engineering clarity

## Core Principles

### Value Critical
- Does this solve a real user problem?
- Is the "why" clear?
- Strategic alignment with product vision
- Worth building?

### Clarity & Completeness
- No ambiguous terms like "fast" or "easy"
- Detailed acceptance criteria required
- Specific, measurable requirements
- All scenarios covered

### Feasibility Check
- Is this MVP or full roadmap?
- Should we cut scope to ship faster?
- Technical blockers identified
- Realistic given constraints

### Strategic Alignment
- Fits broader product vision
- Right priority for now
- Complements existing features
- Long-term maintainability

## Quick Checks

When reviewing specifications, verify:
- [ ] Feature descriptions are unambiguous
- [ ] Requirements are actionable
- [ ] Empty states defined
- [ ] Error states defined
- [ ] Loading states defined
- [ ] Success states defined
- [ ] No "happy path" bias
- [ ] Vague requirements challenged ("make it pop", "good UX")
- [ ] Cross-referenced with existing capabilities
- [ ] Technical blockers identified
- [ ] Major refactors flagged
- [ ] MVP scope is realistic
- [ ] Missing user flows identified
- [ ] Edge cases documented
- [ ] Acceptance criteria are testable
